# Plan: fetch_url 工具（v0.14）

**Date**: 2026-06-21
**Status**: 实施中（多多 6/21 拍板 A + C + E + B-Plus）
**Est. effort**: 1-1.5 天

---

## Goal

让 ACMS 用户在 chat 流里发带 URL 的消息时，自动抓取 URL 正文，**不阻塞 AI 回答**，
**用户可主动选是否沉淀到项目知识库**。架构上走 v0.13 Tool Registry 框架，
未来 AI 可自主调 fetch_url tool。

---

## UX（多多拍板 A + C + E）

- **A**：抓取结果以「📎 参考资料」卡显示在 chat 流用户消息下方
- **C**：不阻塞 AI 回答（抓取 10-30s 进行中时用户消息立即发出，AI 下一轮引用）
- **E**：失败不中断对话（toast + AI 仍正常答）

外加**「📚 加入项目知识库」按钮**（v0.9 promote 模式，用户主动选）

---

## 架构（多多拍板 B-Plus：走 Tool Registry）

- 新 tool `fetch_url`（server/tools/url-fetch.js）
- 新端点 `POST /api/chat/send-with-fetch`（server/routes/chat-fetch.js）
  - 模式 B「预搜注入」：server 内部调 fetch_url tool → 抓取结果入 chat 流 system message → 触发 brief 重生
  - **不传 tools 给 callLLM**（避免 jsonMode 互斥）
- 客户端 regex 检测 + 「📎 参考资料」卡 + 「加入知识库」按钮

未来 AI 自主调 fetch_url（用户说「再读个链接」）走模式 A 纯 tool_calls — 与本路径独立。

## Implementation steps

1. **npm 装 @extractus/article-extractor**（智能正文提取，~100KB）
2. **写 url-safety.js**（SSRF check）
3. **写 url-fetch.js tool**（handler 调 fetch + extractus + 截断）
4. **注册到 tools/index.js**
5. **写 chat-fetch.js 路由**（send-with-fetch 端点 + 预搜注入）
6. **注册到 app.js**
7. **改 chat.js 客户端**：
   - 发送前 regex 检测 URL
   - 显示「🌐 抓取中」状态卡
   - 成功后变「📎 参考资料」卡（含「📚 加入项目库」按钮 + 「× 关闭」）
   - 调 `/api/chat/upload/:id/promote` 落知识库
8. **更新 index.html**（chat-fetch.js script tag）
9. **验证**：
   - node --check 全 14 文件
   - npm test（路由检查）
   - 手测：发带 URL 消息 → 抓取 → 状态卡 → 「加入库」按钮

## Risks

- **R1**: @extractus/article-extractor 内部用 jsdom（~4MB），跟"轻量"原则有冲突
  - 缓解：先装看实际包大小，>500KB 换 cheerio + 自写 Readability（200 行）
- **R2**: SSRF 防护漏（IPv6 / DNS rebinding / URL 编码绕过）
  - 缓解：use `url.parse` + `dns.lookup` 解析所有 IP 后再判断
- **R3**: 反爬（User-Agent 拦截 / Cloudflare challenge）
  - 缓解：失败返回 `{error: 'BLOCKED'}`，不重试
- **R4**: 抓取结果落 chat 流污染 system prompt
  - 缓解：截断 5000 字 + 摘要 + 「▸ 展开」按钮
