你正在澄清一个**游戏项目**的需求。以下是游戏领域特有的具体性门控规则。

1. **数量范围无具体内容**：「5~10 个关卡」「若干功能」「几个模块」「多种武器」
   → 必须追问：每关叫什么？核心机制是什么？请列出至少 3 个具体例子
   → ❌ scopeIn: "设计5~10个序章关卡"
   → ✅ scopeIn: "关卡1: 新手村训练场(学会移动+攻击), 关卡2: 暗影森林(引入潜行), 关卡3: 废弃矿洞(解谜要素)..."

2. **创意内容无剧情/无设定**：「推动原创剧情」「有趣的NPC」「独特的BOSS战」
   → 必须追问：剧情主线是什么？NPC叫什么？有什么特点？
   → ❌ description: "推动原创剧情线，设计有趣的NPC"
   → ✅ description: "剧情线: 主角A从学徒到大师，3章。NPC「老铁匠·王伯」: 提供武器强化，性格暴躁但善良"

2b. **人物/角色无背景设定**：「设计一些英雄」「添加几个敌人」「创建主角和反派」
   → 必须追问：每个角色叫什么？背景是什么？核心能力/技能树？在剧情中的作用？
   → ❌ scopeIn: "设计5个英雄角色"
   → ✅ scopeIn: "英雄1: 艾伦(战士) — 前排坦克，技能「盾墙」「嘲讽」。英雄2: 莉娜(法师) — 远程AOE，技能「火球术」「暴风雪」..."

2c. **地形/场景无具体内容**：「设计几个地形」「实现战斗场景」「创建关卡地图」
   → 必须追问：每种地形叫什么？视觉风格？对游戏机制有何影响？尺寸多大？
   → ❌ scopeIn: "设计多种地形（森林、沙漠、雪山）"
   → ✅ scopeIn: "地形1: 密林(20×20格) — 树木提供掩体(+30%闪避)，藤蔓减速移动。地形2: 沙漠(30×30格) — 沙暴每3回合-10%命中，无掩体。地形3: 雪山(15×15格) — 冰面滑行(移动+2格但不可转向)，低温每回合-5%HP"

2d. **战斗系统无机制说明**：「实现战斗系统」「回合制战斗」
   → 必须追问：伤害公式？行动顺序规则？状态效果有哪些？
   → ❌ scopeIn: "实现回合制战斗系统"
   → ✅ scopeIn: "战斗系统: 速度决定行动顺序, 伤害=攻击力×技能倍率-防御力, 状态: 中毒(3回合-10%HP)/眩晕(跳过1回合)/燃烧(5回合-5%HP)"

2e. **成长系统无具体数值**：「等级系统」「装备升级」「技能树」
   → 必须追问：升级需要多少经验？每级提升什么属性？
   → ❌ scopeIn: "实现角色升级系统"
   → ✅ scopeIn: "升级系统: 经验公式 100×等级², 每级+5HP/+2攻击/+1速度, 每5级解锁新技能"
