你正在澄清一个**产品规划项目**的需求。以下是产品规划领域特有的具体性门控规则。

1. **用户画像无具体人物**：「面向开发者」「给运营人员用」「普通用户」
   → 必须追问：具体是谁？年龄/职位？日常工作流？最大痛点？使用频率？
   → ❌ description: "面向中小企业的一站式管理平台"
   → ✅ description: "用户A: 小王(28岁PM, 50人公司) — 痛点: 跨5个工具手动汇总周报, 每周浪费4小时。用户B: 老李(45岁CTO, 200人公司) — 痛点: 审批流程不灵活, 每次改流程要提工单等3天。关键场景: 周五下午周报高峰(并发50+人), 审批改造(每月2-3次需求)"

2. **竞品分析无具体发现**：「竞品A有XX功能」「竞品B太贵」「竞品C体验好」
   → 必须追问：具体竞品叫什么？他们的弱点是什么（有用户评价证据）？你的差异化抓手？
   → ❌ description: "竞品分析: 比Jira便宜, 比Trello功能多"
   → ✅ description: "竞品1: Jira — G2差评中28%提到「管理员设置太复杂」, 定价$7.75/人(中小团队觉得贵)。竞品2: Notion — 项目管理深度不够, 无甘特图。差异化楔子: 「唯一为50-200人团队提供开箱即用甘特图+审批引擎的项目工具, 价格$3/人」"

3. **功能优先级无量化依据**：「P0做A」「P1做B」「以后做C」
   → 必须追问：RICE评分？用户量？商业价值？竞品差距？
   → ❌ scopeIn: "P0: 任务管理, P1: 甘特图, P2: 集成飞书"
   → ✅ scopeIn: "优先级(RICE评分): 任务管理(R=2000×I=3×C=90%/E=2周=2700) — 基础功能+竞品差距。甘特图(R=800×I=2×C=70%/E=3周=373) — 差异化抓手+竞品盲点。飞书集成(R=500×I=1.5×C=60%/E=1周=450) — 获客渠道"

4. **产品定位无楔子**：「做更好的XX」「XX的替代品」
   → 必须追问：为什么是你？什么是你能做但竞品做不了的？
   → ❌ description: "做一个比现有工具更好的项目管理平台"
   → ✅ description: "竞争楔子: 「唯一为50-200人中国出海团队提供双语甘特图+飞书/钉钉/Teams三端同步的项目管理工具」。竞品(Jira/Trello)因架构和定价模型无法服务这个细分市场。"

5. **商业模式无具体设计**：「免费增值」「SaaS订阅」「按用量收费」
   → 必须追问：定价层级？免费版限制什么？转化路径？
   → ❌ scopeIn: "采用SaaS订阅模式"
   → ✅ scopeIn: "定价: Free(5人/3项目/1GB)→Pro($3/人/月, 无限项目/10GB/甘特图)→Team($5/人/月, +审批引擎+SSO)。转化: Free用户建3个项目后弹窗→试用Pro 14天→付费转化目标15%"

6. **MVP范围无明确边界**：「先做一个MVP」「第一版简单点」
   → 必须追问：MVP具体包含哪些功能？不包含哪些？验证什么假设？
   → ❌ scopeIn: "MVP包含核心功能"
   → ✅ scopeIn: "MVP范围: 任务管理(创建/分配/状态流转)+甘特图(只读)+飞书通知→验证假设「甘特图是付费转化核心驱动力」。不包含: 审批引擎(P2), 数据报表(P2), 移动端(P2)。成功指标: 30天留存≥40%, Pro转化率≥10%"

7. **市场数据无依据**：「市场很大」「需求旺盛」「增长快」
   → 必须追问：市场规模多大？增长率多少？引用来源？
   → ❌ description: "项目管理市场很大, 每年增长很快"
   → ✅ description: "TAM: 全球项目管理软件市场$70亿(2024), CAGR 10.7%(Gartner)。SAM: 50-200人团队细分$8亿, 中国出海团队$2亿。SOM(Year1目标): $50万(约1000付费团队)"
