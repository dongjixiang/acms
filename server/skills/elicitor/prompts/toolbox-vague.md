# 具象化工具箱 — 帮「说不细」的用户落到细节

## 方法论

用户知道自己要什么方向，但说不具体。

## 可选方法（按需选 1-2 个，不要全用）

### 场景压缩
请用户用一句话描述最痛的三秒场景。

引导语：
> 「用一句话告诉我，这个方案解决的是哪个『就那三秒钟』的场景？」

### 极端对比
让用户在最关键的维度上做取舍。

引导语：
> 「如果必须牺牲一项：速度、颜值、成本，你放弃哪个？为什么？」

可以从用户的原始描述中提取 3 个质对，让他选。

### 视觉锚点
如果系统配置了参考图，展示 3 张风格差异大的图问感觉。

引导语：
> 「这里有三种风格——你第一感觉哪个最接近你想要的？哪个地方必须改？」

如果没有参考图，用文字描述 3 种风格让用户选。

## 产出

每次用户回答后，提取出一个可量化的边界：

```json
{
  "elicited_boundaries": [
    { "dimension": "速度", "value": "不超过 2 秒", "confidence": 0.8 }
  ]
}
```

每轮输出新发现的 boundary。空的数组表示这一轮没有可量化的产出。

## 原则

- 用户回答后要立刻小结，让他确认「我理解的对吗」
- 对比之后出现矛盾时不要掩盖——标记为 tradeoff_point 留给 solidify 阶段用
- 允许用户反悔，用新的 answer 覆盖旧的
