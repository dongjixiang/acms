# 逆向筛选工具箱 — 帮「矛盾想法」的用户做减法

## 方法论

用户脑子里有好几个方向在打架，需要帮他排除。

## 可选方法（按需选 1-2 个）

### 反向清单
列出这个方案「绝对不能出现」的特点。

引导语：
> 「这个方案绝对不许出现的三个特点是什么？想到什么说什么。」

不用排序，不用论证，先列出来。

### 失败预演
让用户想象失败场景，反向找出关键约束。

引导语：
> 「如果这个项目半年后失败了，你觉得最可能的原因是什么？」
> 「——那我们怎么避免它？」

### 替身视角
让用户跳出自己的立场。

引导语：
> 「假设你最好的朋友遇到了同样的场景，你会建议他优先关注什么？」

## 产出

```json
{
  "elicited_boundaries": [
    { "dimension": "业务安全", "value": "绝对不依赖第三方登录", "confidence": 0.9 }
  ],
  "eliminated_options": [
    "方向A（XX方案）": "因为用户关注点不在那里"
  ]
}
```

## 原则

- 矛盾本身就是产出——用户表达出「我既想要 X 又想要 Y」时，标记为 tradeoff
- 不要试图替用户做选择，给他呈现矛盾让他自己意识到
- 每个方法结束后问一句「我总结得对吗」再继续
